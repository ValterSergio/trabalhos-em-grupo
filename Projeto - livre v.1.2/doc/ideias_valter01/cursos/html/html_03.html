Estrutura de um documento WEB

A organização de um projeto web é fundamental para a sua manutenção e escalabilidade. Uma estrutura de pastas clara ajuda a manter os arquivos organizados e facilita a colaboração em equipe. A forma mais comum de organizar um projeto HTML é por meio de diretórios dedicados a cada tipo de arquivo.

Arquivo Principal

O arquivo principal de um site é o index.html. Ele geralmente fica na raiz do projeto e serve como a página inicial, sendo o primeiro arquivo que o navegador procura e exibe.

Pastas do projeto

Para manter o projeto arrumado, é uma boa prática criar pastas específicas para os diferentes tipos de recursos.
A pasta img/ é usada para armazenar todas as imagens do seu site, como fotos, ícones e logotipos.
A pasta css/ é onde você salva os arquivos de estilo (".css"), que controlam a aparência do seu site.
Por fim, a pasta js/ é dedicada aos arquivos de script em JavaScript (".js"), que adicionam interatividade e comportamento dinâmico às páginas.
Essa estrutura simples e intuitiva evita que todos os arquivos fiquem misturados na raiz do projeto, tornando mais fácil encontrar, editar e referenciar cada recurso.
